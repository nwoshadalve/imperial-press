# Database ŌĆö backend

## Overview

PostgreSQL 18 accessed via SQLAlchemy 2.0 async ORM. All schema changes go through Alembic migration files.

## Session Management

The async session is injected via `Depends(get_db)` in route functions. Never create sessions manually inside a service or route.

```python
from app.core.database import get_db
from sqlalchemy.ext.asyncio import AsyncSession

@router.get("/{id}")
async def handler(id: uuid.UUID, session: AsyncSession = Depends(get_db)):
    ...
```

The `get_db` dependency context manager commits and closes the session after the request. Service functions do not call `session.commit()` at the end of every function **except** when they need to commit before triggering a side-effect (e.g. commit the status change before generating the certificate PDF).

## Primary Keys

All PKs are **UUIDs**, not sequential integers. This is non-negotiable ŌĆö sequential IDs allow submission enumeration, certificate enumeration, and user enumeration attacks.

```python
import uuid
from sqlalchemy.dialects.postgresql import UUID
from sqlalchemy.orm import Mapped, mapped_column

id: Mapped[uuid.UUID] = mapped_column(
    UUID(as_uuid=True),
    primary_key=True,
    default=uuid.uuid4,
)
```

## Atomic Counter Increments

`view_count` and `download_count` on `Paper` must be updated atomically. Never read the current value and write back ŌĆö this is a race condition under concurrent requests.

```python
# Good ŌĆö atomic UPDATE
from sqlalchemy import update

await session.execute(
    update(Paper)
    .where(Paper.id == paper_id)
    .values(view_count=Paper.view_count + 1)
)
await session.commit()

# Bad ŌĆö read-modify-write (race condition)
paper = await session.get(Paper, paper_id)
paper.view_count += 1
await session.commit()
```

## Migrations (Alembic)

All schema changes go through Alembic. Never run `ALTER TABLE` directly. Never call `Base.metadata.create_all()` in production code.

```bash
# Generate a migration after changing models.py
uv run alembic revision --autogenerate -m "add paper keywords column"

# Apply migrations
uv run alembic upgrade head

# Rollback one step
uv run alembic downgrade -1
```

**Always read the autogenerated migration file before applying it.** Alembic can miss certain changes (type changes, check constraints, partial indexes) or generate an incorrect `drop_table` if a model was renamed. Review every generated file.

## Relationships

Define relationships with `relationship()` and explicit `back_populates`. Never rely on lazy loading ŌĆö use `selectinload` or `joinedload` in repository queries to load related data in a single query.

```python
from sqlalchemy.orm import selectinload

result = await session.execute(
    select(Submission)
    .options(
        selectinload(Submission.files),
        selectinload(Submission.contributors),
    )
    .where(Submission.id == submission_id)
)
```

## Status Enums

Use Python `str` enum for all status columns. The `str` mixin ensures the value serialises as a plain string in JSON responses without extra conversion.

```python
import enum

class SubmissionStatus(str, enum.Enum):
    draft = "draft"
    submitted = "submitted"
    under_review = "under_review"
    revision_requested = "revision_requested"
    accepted = "accepted"
    rejected = "rejected"
    published = "published"
```

Define the enum in `models.py` of the owning domain. Import from there in schemas and services.

## Audit Trail

`Submission` maintains a `SubmissionStatusHistory` table. Every time the submission status changes, the service layer must insert a history row ŌĆö never update `status` without recording the transition.

## Naming Conventions

| Thing | Convention | Example |
|---|---|---|
| Table name | snake_case plural | `submissions`, `review_assignments` |
| Column name | snake_case | `created_at`, `journal_id` |
| FK column | `<singular_related_table>_id` | `submission_id`, `author_id` |
| Index | `ix_<table>_<column>` | `ix_papers_journal_id` |
| Unique constraint | `uq_<table>_<column>` | `uq_users_email` |
| Enum type | PascalCase | `SubmissionStatus`, `CertificateType` |
